# CLAUDE.md

このファイルはClaude Codeがこのプロジェクトで作業する際のガイドラインを定義します。

## プロジェクト概要

アンパンマンのキャラクターを指定すると、そのキャラクターがアニメの何話に登場するかを検索できるWebアプリケーション。

- **フロントエンド**: Next.js 14 (App Router) + TypeScript + Tailwind CSS
- **データソース**: [アンパンマンキャラDB](https://tgws.plus/anpandb/) をスクレイピング

## Git 運用ルール

### 基本方針

- **コードを変更するたびに必ずGitHubへプッシュすること**
- コミットはこまめに行い、変更の単位を小さく保つ

### コミット手順

1. 変更をステージングする
   ```
   git add <変更したファイル>
   ```
2. 意味のあるコミットメッセージでコミットする
   ```
   git commit -m "変更内容の要約"
   ```
3. **即座にGitHubへプッシュする**
   ```
   git push origin <ブランチ名>
   ```

### コミットメッセージの規則

- 日本語・英語どちらでも可
- 変更の「何を」「なぜ」したかを明確に記述する
- 例: `feat: ユーザー認証機能を追加`、`fix: ログインエラーを修正`

### ブランチ戦略

- `main`: 本番相当の安定ブランチ
- 機能追加は feature ブランチを切って作業し、PR経由でマージする
- 直接 main へのプッシュは原則禁止

### 注意事項

- `.env` や秘密情報を含むファイルは絶対にコミットしない
- `git add .` や `git add -A` は意図しないファイルを含む恐れがあるため、ファイルを個別に指定する
